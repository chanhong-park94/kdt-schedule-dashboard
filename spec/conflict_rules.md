# 충돌 판정 규칙 (Conflict Rules)

## 1) 기본 판정식

두 세션 A, B가 충돌하려면 아래를 모두 만족해야 한다.

- 동일 일자
- 동일 기준키(강의실코드 또는 강사코드)
- 서로 다른 `과정기수`
- 시간 구간 overlap:

```txt
A.start < B.end && B.start < A.end
```

## 2) 판정 기준키

- 강의실 기준: `교육장소(강의실)코드`
- 강사 기준: `훈련강사코드`

즉, 같은 날짜 안에서 기준키별 버킷을 만들고 버킷 내부에서 overlap을 계산한다.

## 3) dedup 규칙

충돌 계산 전, 중복 interval 행은 아래 시그니처로 제거한다.

```txt
(훈련일자, 과정기수, 기준키, 훈련시작시간, 훈련종료시간)
```

중복 제거 목적은 동일 세션의 중복 레코드로 인해 충돌 건수가 과대계산되는 문제를 방지하는 것이다.

## 4) 충돌 결과 필드 정의

| 필드명 | 설명 |
|---|---|
| 기준 | `강의실` 또는 `강사` |
| 일자 | 충돌 발생 일자 (`YYYYMMDD`) |
| 키 | 기준키 값(강의실코드/강사코드) |
| 과정A | 첫 번째 기수 |
| A시간 | 첫 번째 기수 시간 구간 (`HHMM-HHMM`) |
| A교과목 | 첫 번째 기수 교과목 코드 |
| 과정B | 두 번째 기수 |
| B시간 | 두 번째 기수 시간 구간 (`HHMM-HHMM`) |
| B교과목 | 두 번째 기수 교과목 코드 |

표시는 상위 200건으로 제한하되, 총 충돌 건수는 별도 카운트로 표시한다.

## 5) 향후 확장: 화이트리스트/예외룰

향후 아래 확장 포인트를 지원할 수 있다.

- 화이트리스트: 특정 강사/강의실/기간/시간대는 충돌에서 제외
- 예외룰 우선순위: `예외룰 > 기본 충돌식`
- 예외 사유 기록: 충돌 제외 근거(`reason`, `updatedBy`, `updatedAt`) 저장
- 시뮬레이션 모드: 예외룰 적용 전/후 충돌 건수 비교
