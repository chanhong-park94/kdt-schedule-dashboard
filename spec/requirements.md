# 요구사항 명세 (Requirements)

## 기능 범위

### In Scope
- 세션 마스터 CSV 업로드 및 파싱
- `과정기수` 목록 자동 생성 및 선택
- 선택한 기수의 기간/훈련일수/세션수 요약 표시
- 충돌 탐지(기준: 강의실/강사, 동일 일자, 시간 구간 overlap)
- HRD 업로드용 CSV 다운로드(9개 컬럼, `과정기수` 제외)

### Out of Scope
- 서버/DB/인증 등 백엔드 기능
- 세션 데이터 직접 수정/저장 기능
- 외부 시스템 자동 동기화(API 연동)
- 권한 관리/다중 사용자 협업
- 화이트리스트/예외룰의 고급 정책 UI

## 사용자 시나리오 (3)

1. 운영 담당자는 세션 마스터 CSV를 업로드하고, 자동 생성된 기수 목록에서 특정 `과정기수`를 선택해 요약 정보를 확인한 뒤 HRD 업로드용 CSV를 내려받는다.
2. 스케줄 관리자는 충돌 목록(강의실/강사 기준)을 확인하고, 소스 CSV를 수정한 후 재업로드하여 충돌 건수가 줄었는지 재검증한다.
3. QA 담당자는 중복 행이 포함된 CSV를 업로드해 dedup 로직이 동작하는지 확인하고, 동일 데이터 재업로드 시 일관된 결과가 나오는지 검증한다.

## 에러 케이스/제약 (5)

1. 필수 컬럼 누락: 스키마에 정의된 컬럼명이 다르거나 누락되면 정상 계산이 불가하다.
2. 날짜/시간 형식 오류: `YYYYMMDD` 또는 `HHMM` 형식이 아니면 시간 계산 및 충돌 탐지에서 제외될 수 있다.
3. 시간 역전 데이터: 시작시간이 종료시간보다 크거나 같은 행은 유효한 interval로 처리하지 않는다.
4. 빈값/NA 데이터: 강사코드/강의실코드/일자 등 기준값이 비어 있으면 해당 행은 충돌 판정 대상에서 제외된다.
5. 대용량 파일 제약: 화면에는 충돌 상위 1,000건만 표시하며, 전체 건수는 별도 카운트로 제공한다.
